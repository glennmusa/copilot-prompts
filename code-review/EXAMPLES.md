# Code Review Examples

## General review

```
Review the selected code. List any bugs, logic errors, security vulnerabilities, and style issues. For each issue, explain why it's a problem and suggest a fix.
```

## Security-focused review

```
Audit the selected code for security vulnerabilities — including injection risks, authentication flaws, insecure defaults, and sensitive data exposure. Explain each finding and how to remediate it.
```

## Performance review

```
Review the selected code for performance issues. Identify any inefficient algorithms, unnecessary allocations, blocking calls, or missed caching opportunities. Suggest concrete improvements.
```

## Pull request readiness check

```
Pretend you are a senior engineer reviewing this code before merging. What questions would you ask? What would you ask the author to change? What looks good?
```

## Readability and maintainability

```
Review the selected code for readability and maintainability. Flag anything that would slow down a new contributor trying to understand this code, and suggest clearer alternatives.
```
