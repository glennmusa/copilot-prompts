# Write Docs Examples

## Add docstrings

```
Add docstrings to every function and class in the selected code. Follow the documentation style already used in this project (e.g., JSDoc, Google-style, NumPy-style).
```

## Write a README for a module

```
Write a README.md for #file. Include: what it does, who should use it, how to install and configure it, a quick-start example, and a reference of its public API.
```

## Add inline comments

```
Add inline comments to the selected code to explain non-obvious logic. Don't comment what the code does — comment why it does it that way.
```

## Document a REST API

```
Write API documentation for the selected route handlers. For each endpoint include: method, path, description, request parameters/body, response shape, and error responses.
```

## Write a changelog entry

```
Write a changelog entry for the changes in this pull request. Follow the Keep a Changelog format. Categorize changes as Added, Changed, Deprecated, Removed, Fixed, or Security.
```

## Generate a usage guide

```
Write a usage guide for the selected code targeting a developer who has never used it before. Include a brief overview, prerequisites, step-by-step setup, and two or three practical examples.
```
