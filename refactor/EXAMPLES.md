# Refactor Examples

## General cleanup

```
Refactor the selected code to improve readability and maintainability. Do not change observable behavior. Explain each change you make.
```

## Extract functions

```
Refactor the selected code by extracting logical sections into well-named helper functions. Keep each function focused on a single responsibility.
```

## Remove duplication

```
Identify and eliminate duplication in the selected code. Consolidate repeated logic into shared functions or abstractions without over-engineering.
```

## Simplify conditionals

```
Simplify the conditional logic in the selected code. Replace deeply nested if/else chains with early returns, guard clauses, or cleaner patterns where appropriate.
```

## Apply a design pattern

```
Refactor the selected code to apply an appropriate design pattern. Explain which pattern you chose and why it improves the code.
```

## Improve naming

```
Improve the variable, function, and parameter names in the selected code so they clearly communicate intent without requiring comments.
```

## Reduce function length

```
The selected function is too long. Refactor it into smaller, focused functions. Each function should do one thing and be easy to test in isolation.
```
